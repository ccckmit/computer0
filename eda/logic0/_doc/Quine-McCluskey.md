# Quine-McCluskey Method (QM Method)

## Overview

Quine-McCluskey (QM) Method 是一種用於化簡布林代數表達式的系統化方法，又稱為**質項圖法**或**Tabular Method**。由 Willard Van Orman Quine 和 Edward J. McCluskey 分別於 1952 年及 1956 年提出。該方法彌補了卡諾圖 (Karnaugh Map) 在變數數量過多時難以手動操作的缺陷，可機械化地執行，適合實現為電腦程式。

## 核心概念

### 質項 (Prime Implicant)

一個**質項**是指無法再進一步與其他項合併的蘊含項 (Implicant)。換言之，該蘊含項無法與另一個僅相差單一位元的蘊含項合併為具有更少常值的項。

### 必要質項 (Essential Prime Implicant)

若某個質項**唯一涵蓋**某個最小項 (minterm)，則該質項為必要質項。這些必要質項必定會出現在最終的最簡式中。

### 覆蓋問題

給定一組最小項，找出能涵蓋所有最小項的最小質項集合，這是一個集合覆蓋問題。

## Method 步驟

### Step 1: 將最小項依據 1 的個數分組

將所有需要涵蓋的最小項 (minterms) 根據其二進位表示式中 `1` 的個數進行分組。

**範例**: 最小項 m(0, 1, 2, 3, 5, 7, 8, 10, 13)

| 最小項 | 二進位 (A B C D) | 1 的個數 |
|--------|-----------------|----------|
| 0      | 0000            | 0        |
| 1      | 0001            | 1        |
| 2      | 0010            | 1        |
| 8      | 1000            | 1        |
| 3      | 0011            | 2        |
| 5      | 0101            | 2        |
| 10     | 1010            | 2        |
| 7      | 0111            | 3        |
| 13     | 1101            | 3        |

### Step 2: 第一輪比較 (Pairing)

比較相鄰組（1 的個數相差 1 的組），找出可以合併的項。合併規則：兩個只有**單一位元不同**的項可以合併，合併後該位元以 `-` (dash) 表示，表示該位元為 don't care。

合併時，檢查是否已比較過，僅標記未曾比較過的項。

**範例**:

- `0 (0000)` 與 `1 (0001)` 合併 → `0,1 (000-)`
- `0 (0000)` 與 `2 (0010)` 合併 → `0,2 (00-0)`
- 依此類推...

### Step 3: 重複比較直到無法再合併

持續對上一輪產生的結果進行比較，直到沒有新的合併發生。所有**未被合併**的項即為**質項**。

### Step 4: 製作質項覆蓋表

建立一個表格，橫軸為所有原始最小項，縱軸為所有質項，標記每個質項涵蓋了哪些最小項。

### Step 5: 找出必要質項

若某個最小項**僅被一個質項涵蓋**，則該質項為必要質項，將其加入最終答案，並移除該質項所涵蓋的所有最小項。

### Step 6: 簡化覆蓋表

移除必要質項及其涵蓋的最小項後，檢查是否還有需要處理的行：
- 若只剩一行，選擇涵蓋該行的任一質項
- 若存在**列主導**關係（某列純粹涵蓋另一列涵蓋的最小項的子集），可移除被支配的列
- 重複 Step 5-6 直到完成

### Step 7: 選擇剩餘質項

使用以下策略之一選擇剩餘質項：
- **Petrick's Method**: 將問題轉換為標準合取範式 (CNF) 求解
- **Branching Method**: 貪心選擇或嘗試分支

## 完整範例

### 問題

化簡布林函數:
```
F(A, B, C, D) = Σm(0, 1, 2, 3, 5, 7, 8, 10, 13)
```

### Step 1-3: 分組與合併

經過三輪合併後得到以下質項:

| 質項 | 二進位表示 | 涵蓋最小項 |
|------|-----------|-----------|
| P1   | 00--      | 0, 1, 2, 3 |
| P2   | -001      | 1, 5      |
| P3   | 01-1      | 5, 7      |
| P4   | -010      | 2, 10     |
| P5   | -101      | 10, 13    |
| P6   | 1-01      | 8, 13     |
| P7   | 1-0-      | 8, 10     |

### Step 4: 質項覆蓋表

| 質項 \ 最小項 | 0 | 1 | 2 | 3 | 5 | 7 | 8 | 10 | 13 |
|-------------|---|---|---|---|---|---|---|----|----|
| P1 (00--)   | X | X | X | X |   |   |   |    |    |
| P2 (-001)   |   | X |   |   | X |   |   |    |    |
| P3 (01-1)   |   |   |   |   | X | X |   |    |    |
| P4 (-010)   |   |   | X |   |   |   |   | X  |    |
| P5 (-101)   |   |   |   |   |   |   |   | X  | X  |
| P6 (1-01)   |   |   |   |   |   |   | X |    | X  |
| P7 (1-0-)   |   |   |   |   |   |   | X | X  |    |

### Step 5: 找出必要質項

- 最小項 0: 只被 P1 涵蓋 → **P1 為必要質項**
- 最小項 3: 只被 P1 涵蓋 → 已被 P1 涵蓋
- 最小項 7: 只被 P3 涵蓋 → **P3 為必要質項**
- 最小項 8: 被 P6 和 P7 涵蓋

選擇必要質項 P1 和 P3，移除已涵蓋的最小項 (0, 1, 2, 3, 5, 7)。

### Step 6-7: 處理剩餘

移除已涵蓋最小項及相關標記後:

| 質項 \ 最小項 | 8  | 10 | 13 |
|-------------||----|----|----|
| P5 (-101)   |    | X  | X  |
| P6 (1-01)   | X  |    | X  |
| P7 (1-0-)   | X  | X  |    |

使用 Petrick's Method 或分支法選擇:

- 選擇 P6 (1-01) 涵蓋 8 和 13
- 選擇 P4 (-010) 涵蓋 10（原本 P4 已被 P7 替代，但 P7 未涵蓋 10）

### 最終結果

```
F = P1 + P3 + P6 + P4
  = 00-- + 01-1 + 1-01 + -010
  = A'B' + AC'D + AD' + B'D
```

## 與卡諾圖的比較

| 特性         | 卡諾圖 (K-map)      | Quine-McCluskey      |
|------------|---------------------|----------------------|
| 適用變數數量 | 2-6 個變數          | 無理論上限           |
| 視覺化       | 有                  | 無                   |
| 手動操作     | 直覺快速            | 系統化但繁瑣         |
| 自動化       | 難以程式化          | 容易程式化           |

## 局限性

1. **記憶體爆炸**: 質項數量隨輸入變數數量指数成長
2. **NP 完全問題**: 找到最小覆蓋的時間複雜度為指數級
3. **不保證絕對最小**: 在某些情況下，Petrick's Method 可能無法找到絕對最小解
